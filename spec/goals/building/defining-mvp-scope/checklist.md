# Checklist: Defining MVP scope

Tick when done. Adapt wording to your stage.

- [ ] Single primary user job in scope; non-goals written down
- [ ] Smallest artifact that can test the hypothesis is identified
- [ ] Explicit out-of-scope list agreed with product and eng
- [ ] Success criteria for 'learn enough' are defined, not infinite polish
- [ ] Timeline and owner for the scoped slice are named
- [ ] Link to the next ship milestone (e.g. shipping-mvp) is clear
